Diagnóstico del Bug

  Causa raíz — NO está en validarCargaDocente

  validarCargaDocente SÍ funciona y devuelve la carga correctamente. El problema real está en getCompetenciasConCriterios en CriterioModel.php.

  El problema: inconsistencia en la columna subarea_id / area_id de competencias

  El query UNION tiene dos ramas:

  -- Rama 1: competencia vinculada por subarea_id
  ON ca.subarea_id = comp.subarea_id  WHERE comp.subarea_id IS NOT NULL

  -- Rama 2: competencia vinculada por area_id
  ON ca.area_id = comp.area_id  WHERE comp.area_id IS NOT NULL

  El conflicto: Las cargas de secundaria con subáreas (ej. docente_id=7, Literatura, carga_id=1) tienen subarea_id=15 y area_id=NULL. Pero las
  competencias de secundaria (C38, C39, C40 de Comunicación) están guardadas con area_id=18 y subarea_id=NULL.

  ┌────────────────────────────────────────────────┬────────────┬─────────┐
  │                     Tabla                      │ subarea_id │ area_id │
  ├────────────────────────────────────────────────┼────────────┼─────────┤
  │ cargas_academicas (docente 7, Literatura)      │ 15         │ NULL    │
  ├────────────────────────────────────────────────┼────────────┼─────────┤
  │ competencias (C38, C39, C40 Comunicación sec.) │ NULL       │ 18      │
  └────────────────────────────────────────────────┴────────────┴─────────┘

  Rama 1: 15 ≠ NULL — no coincide
  Rama 2: NULL ≠ 18 — no coincide
  → $competencias = [] → la vista muestra vacío

  Las competencias de primaria SÍ tienen subarea_id correcto (ej. C21 tiene subarea_id=4). El problema es exclusivo de secundaria con_subareas.

  Áreas afectadas (todo el nivel secundaria con_subareas)

  - Ciencias Sociales (area_id=17): C30, C31, C32 deberían tener subarea_id 11,12,13
  - Comunicación (area_id=18): C38, C39, C40 → subareas 14,15,16
  - Matemática (area_id=19): C44, C45, C46, C47 → subareas 17,18,19,20
  - Ciencia y Tecnología (area_id=20): C48, C49, C50 → subareas 21,22,23

  Por qué tu prueba con docente_id=7 no devolvía resultados

  Probablemente ejecutaste el query de validarCargaDocente sin sustituir el primer ? (ca.id), o usaste un carga_id incorrecto. El query en sí
  funciona bien para la carga correcta.
